# Migrating to v17.0.0

v17.0.0 is a major release. The entire source tree is now TypeScript.
The public API has a new entry point (`openWarpGraph`), symbol renames,
deleted modules, and changed import paths. This guide covers every
breaking change and provides migration scripts to automate the
mechanical work.

## Quick start

Run the automated migration script first, then address any remaining
manual changes:

```bash
# From your project root (the one that depends on @git-stunts/git-warp)
npx @git-stunts/git-warp@17 migrate-imports
```

Or run the scripts individually:

```bash
# 1. Fix import paths (.js → .ts, moved files)
node node_modules/@git-stunts/git-warp/scripts/migrations/v17.0.0/fix-imports.js

# 2. Fix symbol renames
node node_modules/@git-stunts/git-warp/scripts/migrations/v17.0.0/fix-renames.js
```

After running the scripts, build your project and fix any remaining
type errors manually.

## Breaking changes (detailed)

### 1. New entry point: `openWarpGraph()`

**Before (v16):**
```typescript
import WarpApp from '@git-stunts/git-warp';

const graph = await WarpApp.open({
  persistence,
  graphName: 'events',
  writerId: 'node-1',
});

await graph.materialize();
const nodes = await graph.getNodes();
```

**After (v17):**
```typescript
import { openWarpGraph } from '@git-stunts/git-warp';

const graph = await openWarpGraph({
  persistence,
  graphName: 'events',
  writerId: 'node-1',
});

await graph.materialize.materialize({});
const nodes = await graph.query.getNodes();
```

The returned `WarpGraph` object has 9 capability namespaces:

| Namespace | Purpose | Key methods |
|-----------|---------|-------------|
| `graph.query` | Read nodes, edges, props, content | `getNodes()`, `getNodeProps()`, `getEdges()`, `neighbors()` |
| `graph.patches` | Create and commit patches | `createPatch()`, `discoverWriters()` |
| `graph.materialize` | Frontier-relative state | `materialize()`, `materializeCoordinate()` |
| `graph.sync` | Distributed sync | `syncWith()`, `syncNeeded()`, `getFrontier()`, `status()`, `serve()` |
| `graph.strands` | Speculative lanes | `createStrand()`, `braidStrand()`, `materializeStrand()` |
| `graph.checkpoint` | History folding | `createCheckpoint()`, `loadCheckpoint()` |
| `graph.provenance` | Witness access | `getProvenanceIndex()`, `verifyAuditChain()` |
| `graph.comparison` | Coordinate comparison | `compareCoordinates()`, `planCoordinateTransfer()` |
| `graph.subscriptions` | Reactive state | `subscribe()`, `unsubscribe()` |

**Backward compatibility:** `WarpApp` and `WarpCore` are still
exported for backward compatibility. They will be removed in v18.

### 2. Import path changes

All source files moved from `.js` to `.ts` extensions. If you import
internal modules directly (not through the barrel `index.js`), update
your paths:

```diff
-import GitGraphAdapter from '@git-stunts/git-warp/src/infrastructure/adapters/GitGraphAdapter.js';
+import GitGraphAdapter from '@git-stunts/git-warp/src/infrastructure/adapters/GitGraphAdapter.ts';
```

**Files that moved directories:**

| Old path | New path |
|----------|----------|
| `src/domain/services/BitmapIndexBuilder.js` | `src/domain/services/index/BitmapIndexBuilder.js` |
| `src/domain/services/BitmapIndexReader.js` | `src/domain/services/index/BitmapIndexReader.js` |
| `src/domain/services/IndexRebuildService.js` | `src/domain/services/index/IndexRebuildService.js` |
| `src/domain/services/CommitDagTraversalService.js` | `src/domain/services/dag/CommitDagTraversalService.ts` |
| `src/domain/services/Observer.js` | `src/domain/services/query/Observer.ts` |
| `src/domain/services/QueryBuilder.js` | `src/domain/services/query/QueryBuilder.ts` |
| `src/domain/services/StateReaderV5.js` | `src/domain/services/state/StateReader.ts` |
| `src/domain/services/StateSerializerV5.js` | `src/domain/services/state/StateSerializer.ts` |
| `src/domain/services/VisibleStateComparisonV5.js` | `src/domain/services/comparison/VisibleStateComparison.ts` |
| `src/domain/services/VisibleStateScopeV1.js` | `src/domain/services/VisibleStateScope.ts` |
| `src/domain/services/ProvenanceIndex.js` | `src/domain/services/provenance/ProvenanceIndex.js` |
| `src/domain/services/ProvenancePayload.js` | `src/domain/services/provenance/ProvenancePayload.ts` |

The migration script handles all of these automatically.

### 3. Symbol renames

| Old name | New name | Reason |
|----------|----------|--------|
| `PatchV2` | `Patch` | V2 suffix dropped |
| `PatchBuilderV2` | `PatchBuilder` | V2 suffix dropped |
| `Lens` | `Aperture` | Observer geometry rename |
| `StateReaderV5` | `StateReader` | V5 suffix dropped |
| `StateSerializerV5` | `StateSerializer` | V5 suffix dropped |
| `VisibleStateComparisonV5` | `VisibleStateComparison` | V5 suffix dropped |
| `VisibleStateTransferPlannerV5` | `VisibleStateTransferPlanner` | V5 suffix dropped |
| `VisibleStateScopeV1` | `VisibleStateScope` | V1 suffix dropped |
| `CheckpointSerializerV5` | `CheckpointSerializer` | V5 suffix dropped |

Old names are available as type aliases in `index.d.ts` for backward
compatibility. They will be removed in v18.

### 4. Deleted modules

| Module | Replacement |
|--------|-------------|
| `src/visualization/` (entire directory) | Use `@git-stunts/warp-ttd` |
| `src/domain/types/WarpTypes.ts` | Op factories inlined in `index.js` |
| `src/domain/types/WarpTypesV2.ts` | Use `Patch`, `PatchBuilder` directly |
| `src/domain/services/StrandService.js` | Use `StrandCoordinator` via `graph.strands` |

### 5. Op type changes

Operations are now frozen class instances with constructor validation:

```diff
-const op = { type: 'NodeAdd', node: 'user:alice', dot: { writerId: 'w1', counter: 1 } };
+import { NodeAdd, Dot } from '@git-stunts/git-warp';
+const op = new NodeAdd('user:alice', new Dot('w1', 1));
```

The old plain-object factory functions (`createNodeAdd`, etc.) are
still exported for backward compatibility but are deprecated.

### 6. Index shard format change

Bitmap index shards changed from JSON envelopes with checksums to
plain CBOR. **No action required** — existing cached indexes are
automatically rebuilt on first materialization. The rebuild is
transparent but may cause a one-time slowdown on large graphs.

### 7. `createEventId` / `createDot` removed

```diff
-import { createEventId, createDot } from '@git-stunts/git-warp';
-const id = createEventId('w1', 5);
-const dot = createDot('w1', 1);
+import { EventId, Dot } from '@git-stunts/git-warp';
+const id = new EventId('w1', 5);
+const dot = new Dot('w1', 1);
```

### 8. Type parameter changes

Several types narrowed from `unknown` to concrete types. If you
were relying on the loose typing, you may need to update call sites:

| API | v16 type | v17 type | Impact |
|-----|----------|----------|--------|
| `materialize()` return | `WarpStateV5` | `WarpState` | Same shape, new name |
| `observer()` config param | `Lens` | `Aperture` | Rename only |
| `hmac()` return | `Uint8Array` | `Uint8Array` | No change (was already Uint8Array) |
| `WarpState.prop` values | `LWWRegister<unknown>` | `LWWRegister<PropValue>` | Narrowed — `PropValue` is `{ type: 'inline', value: unknown } \| { type: 'blob', oid: string }` |
| `WarpState.observedFrontier` | `Map<string, number>` | `VersionVector` | Runtime-backed class |

**`PropValue` narrowing** is the most impactful type change. If you
access `.prop.get(key).value` directly, you now get `PropValue`
instead of `unknown`. Extract the actual value:

```diff
-const color = state.prop.get(key).value;
+const register = state.prop.get(key);
+const color = register?.value.type === 'inline' ? register.value.value : null;
```

Or use the convenience accessor if you know it's inline:

```typescript
const color = (register?.value as { type: 'inline'; value: string }).value;
```

### 9. `materialize()` options object

`materialize()` now requires an options object, even if empty:

```diff
-await graph.materialize();
+await graph.materialize({});
```

When using the new `openWarpGraph()` API:

```diff
-await graph.materialize();
+await graph.materialize.materialize({});
```

### 10. Effect pipeline changes

`PatchBuilder.emitEffect()` replaces the old effect emission pattern:

```diff
-graph.emit('effect', payload);
+const patch = await graph.patches.createPatch();
+patch.emitEffect(payload);
+await patch.commit();
```

## Migration checklist

- [ ] Run `fix-imports.js` migration script
- [ ] Run `fix-renames.js` migration script
- [ ] Run `verify.js` to check for remaining v16 patterns
- [ ] Update `openWarpGraph()` entry point (or keep using `WarpApp.open()` temporarily)
- [ ] Replace direct op object construction with class constructors
- [ ] Replace `createEventId`/`createDot` with `new EventId`/`new Dot`
- [ ] Add `{}` argument to bare `materialize()` calls
- [ ] Update `PropValue` access patterns (`.value` is now typed)
- [ ] Update `Lens` → `Aperture` in observer configs
- [ ] Remove any imports from `src/visualization/` (use warp-ttd)
- [ ] Remove any imports of `WarpTypes` or `WarpTypesV2`
- [ ] Build and fix remaining type errors
- [ ] Run your test suite
