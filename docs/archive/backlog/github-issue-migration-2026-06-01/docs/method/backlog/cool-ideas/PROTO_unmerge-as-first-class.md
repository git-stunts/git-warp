# Unmerge as first-class

If merge becomes a witnessed causal object instead of an opaque event, then
``undo merge'' should stop being folklore.

Potential shape:

- invert a canonical join using stored witness
- reopen an enriched merge object instead of synthesizing a revert
- recover the original branch pair plus obstruction witness for inspection

This would make reversal of merge much cleaner than today's revert-and-pray
story.
