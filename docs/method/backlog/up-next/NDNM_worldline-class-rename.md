# Rename Worldline class to match theory

**Effort:** M

## Problem

The `Worldline` class is not a worldline. In the papers (WARP II,
OG-I §2), a worldline is the deterministic sequence
`U₀ → U₁ → … → Uₙ` generated by ticks under a scheduling policy.

In the codebase, `Worldline` is a thin read-handle that wraps a graph
reference and a viewpoint descriptor. It calls `materialize()` and
returns a snapshot. It has no history, no sequence, no BTR chain.

It is a **projection handle** — the `Π` component of a replica
`R = (S, F, Π)`.

## Fix

Rename to `ReadHandle`, `ProjectionHandle`, or `WorldlineView`.
Exact name TBD at design time.

## Breaking change assessment

`Worldline` is a public API class exported from index.js and declared
in index.d.ts. This is a breaking change requiring a major version
bump.
