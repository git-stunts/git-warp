# Sludge Striker End-of-Turn Protocol

## Idea

Formalize the required end-of-turn `SLUDGE STRIKER SUMMARY` as project
doctrine.

Required sections:

- Sludge encountered
- Sludge fixed
- Sludge rejected
- Sludge deferred / tracked
- Anti-sludge checks actually run
- Remaining risk

## Why It Is Cool

It makes the agent produce a police report instead of a lullaby.

## Guardrails

- This is process doctrine, not a license to make every turn long.
- Keep claims evidence-scoped.
- Do not turn the summary into ceremonial prose.
- If automated later, preserve the human-readable accountability shape.
