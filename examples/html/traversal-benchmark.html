Example 6

traversal-benchmark.js: Weighted Search

Generate linear and diamond graphs, materialize, then benchmark Dijkstra vs A*.

Run it

node scripts/traversal-benchmark.js

Creates synthetic graphs and times pathfinding.

Setup

Generate graphs of different shapes and sizes, then benchmark traversal algorithms against the materialized state.

Run it with node scripts/traversal-benchmark.js to generate graphs and timings.

Git DAG (patch batches)

Each batch of node additions is committed as a patch.

WARP Graph (sample shapes)

Benchmarks compare linear chains vs diamond DAGs.

Explanation

  1. Create a linear graph and a diamond graph for each size.
  2. Commit nodes and edges in batches as patches.
  3. Each batch becomes a commit in the writer ref chain.
  4. Materialize to build the WARP state for each graph.
  5. Build adjacency from edges in the materialized state.
  6. Run Dijkstra and A* multiple times to get median timings.
  7. Measure visited node counts for each traversal shape.
  8. Compare traversal cost across shapes and sizes.
  9. Use the results to tune heuristics and edge weights.
  10. Remember: history is batches, but traversal is on state.

Conclusion

You now have traversal timing intuition. Next, streaming-benchmark.js focuses on ingest throughput.
